# 版本规则

版本：`v16.1.260606.2115`

v16 系列使用四段版本号：

```text
v<major>.<minor>.<YYMMDD>.<HHMM>
```

Python package metadata 不带前缀 `v`：

```text
16.1.260606.2115
```

## 含义

以 `v16.1.260606.2115` 为例：

- `v16`：较大的版本系列。
- `1`：v16 内的小版本。
- `260606`：修改日期，YYMMDD。
- `2115`：coding agent 修改项目的 24 小时时间，HHMM。

## 更新规则

每次 Codex 从 plan 实现修改时，小版本加一，并更新日期和时间。

从一个 v16 实现版本更新到下一个版本的示例：

```text
v16.<old-minor>.<old-date>.<old-time> -> v16.1.260606.2115
```

后续如果同一天 22:04 再次实现计划：

```text
v16.1.260606.2115 -> v16.2.260606.2204
```

## 必须同步的位置

- 程序窗口标题。
- About dialog 的版本信息。
- `pyproject.toml`。
- `src/ca_app/__init__.py`。
- `AGENTS.md`。
- `README.md`。
- `README.zh-CN.md`。
- 英文和中文文档。

当前文件夹名可以保留为复制时的历史名称，不需要每次小版本修改都复制新文件夹，除非用户明确要求。
